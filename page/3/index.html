fastJSON 转泛型对象

fastJSON 转泛型对象

fastJSON在实际的使用过程中，需要将JSON字符串转成带有泛型的对象。普通的方式无法实现。查询相关的资料最后得到的结果如下：

public List<ResInnoIds> callStatByInno(List<Integer> innoIds){
    Map<String,Object> request = new HashMap<>();
    request.put("ccId",inputParamHead.getCcId());
    request.put("vdnId",inputParamHead.getVdnId());
    if(innoIds!=null&&innoIds.isEmpty()){
        request.put("innoIds",innoIds);
    }
    String url = CmsEndPoint.get("real.accesscode.callstatbyinno");
    String response = HttpUtil.post(url,request);
    Response<List<ResInnoIds>> res= JSON.parseObject(response,new TypeReference<Response<List<ResInnoIds>>>(){});
    return res.getResult();
}

由于Response<List<ResInnoIds>>是一个泛型对象。
使用JSON.toJavaObject(json,Response.class）无法将其转成Response>。
必须使用

JSON.parseObject(response,new TypeReference<Response<List<ResInnoIds>>>(){});

Spring注解泛解

Spring注解泛解

1. @Autowired

@Autowired注解用于bean注入与@Resource等效，被注入的bean必须为Spring IOC容器管理的bean。
Spring管理的bean有如下特点：

  1. @Component, @Repository, @Controller, @Service, @Configration这些注解都是把你要实例化的对象转化成一个Bean，放在IoC容器中，等你要用的时候，它会和上面的@Autowired , @Resource配合到一起，把对象、属性、方法完美组装。

2.@Controller

@Controller是控制层声明的注解，该注解声明了本类是控制层，常与@ResponseBody一起使用，主要功能返回一个JSP页面，而如果单独使用@Controller则返回值就为return 的返回值。而在Spring boot中常常使用@RestController。可以简单的理解为@RestController=@ResponseBody+@Controller。

3.@Repository

@Repository注解主要用于dao层（持久化层），声明该类属于数据层，主要用于对数据CRUD操作的层。

4.@Service

@Service 注解主要用于Service层，使用该注解一般为业务层。主要功能实现业务逻辑。

5.@Component

@Component注解主要用于组件注解，如定时器，工具bean等。

6.@RequestMapping

@RequestMapping是一个用来处理请求地址映射的注解，可用于类或方法上。用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径。

7.@PathVariable

@PathVariable是获取路径中的通配符的注解。该注解可以用于Restful请求获取路径中的参数开发。
例子：

@RequestMapping(value = "/history/{method}", method = RequestMethod.POST,consumes="appplication/json")
public Response historyMonitor(@PathVariable @Valid @NotNull(message = "method不能为空") String method, @RequestBody @Valid ParamIndex paramIndex, BindingResult bindingResult) {
    logger.info("入参：method={},paramIndex={}",method,JSON.toJSONString(paramIndex));
    if (bindingResult.hasErrors()) {
        return CheckParam.getErrors(bindingResult);    
    }
    logger.info("params={}", JSON.toJSONString(paramIndex));
    List<String> result = checkIndicator(paramIndex.getIndexIds());
    if(!result.isEmpty()){
        return new Response().setRetcode("-1").setMessage("部分指标不存在！").setResult(result);
    }
    switch (method) {
        case "agent":
            return agentMonitorService.hindexAgent(paramIndex);
        case "ivr":
            return ivrMonitorService.hindexIvr(paramIndex);
        case "skill":
            return skillMonitorService.hindexSkill(paramIndex);
        case "vdn":
            return vdnMonitorService.hindexVdn(paramIndex);
    }
    return new Response().setMessage("访问路径不存在/indicators/history/"+method).setRetcode("-1");
}

8.@Valid

@Valid注解用于校验，所属包为：javax.validation.Valid。

① 首先需要在实体类的相应字段上添加用于充当校验条件的注解，如：@Min,如下代码（age属于Girl类中的属性）：

@Min(value = 18,message = "未成年禁止入内")  
private Integer age;

② 其次在controller层的方法的要校验的参数上添加@Valid注解，并且需要传入BindingResult对象，用于获取校验失败情况下的反馈信息，如下代码：

@PostMapping("/girls")  
public Girl addGirl(@Valid Girl girl, BindingResult bindingResult) {  
    if(bindingResult.hasErrors()){  
        System.out.println(bindingResult.getFieldError().getDefaultMessage());  
        return null;  
    }  
    return girlResposity.save(girl);  
}

bindingResult.getFieldError.getDefaultMessage() 用于获取相应字段上添加的message中的内容，如：@Min注解中message属性的内容
